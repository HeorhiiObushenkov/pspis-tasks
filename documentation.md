# Документація проєкту "Семестровий Дашборд"

## 1. Огляд проєкту
"Семестровий Дашборд" — це веб-додаток для студентів групи ПЗПІз-25-1, призначений для організації та відстеження навчальних матеріалів, завдань та прогресу з різних дисциплін. Додаток побудований на сучасних веб-технологіях та забезпечує швидкий доступ до інформації через інтуїтивно зрозумілий інтерфейс.

## 2. Технологічний стек
- **HTML5 & CSS3**: Структура та стилізація.
- **Tailwind CSS (v3)**: JavaScript-перший CSS фреймворк для швидкої розробки дизайну.
- **DaisyUI (v4)**: Плагін для Tailwind CSS, що надає набір готових компонентів (модальні вікна, картки, меню).
- **Vanilla JavaScript (ES Modules)**: Логіка додатку без сторонніх важких фреймворків.
- **JSON**: Зберігання даних про дисципліни та структуру контенту.
- **LocalStorage**: Збереження стану (виконані завдання, вибрані теми).

## 3. Структура проєкту
- `index.html`: Основний файл розмітки, точка входу в додаток.
- `style.css`: Додаткові кастомні стилі.
- `js/`: Директорія з логікою на JavaScript.
  - `main.js`: Ініціалізація додатку, маршрутизація (hash-based), обробка налаштувань.
  - `render.js`: Модуль для візуалізації UI-компонентів та віджетів.
  - `state.js`: Управління глобальним станом додатку (кеш, завдання).
  - `theme.js`: Логіка перемикання світлої та темної тем.
  - `icons.js`: Бібліотека SVG-іконок.
- `data/`: Дані додатку.
  - `manifest.json`: Список доступних дисциплін та шляхи до їх файлів.
  - `[subject].json`: Контент конкретної дисципліни.

## 4. Схема даних
### Manifest (`manifest.json`)
```json
{
    "subjects": [
        {
            "id": "subject-id",
            "name": "Назва дисципліни",
            "path": "./data/subject.json",
            "icon": "IconName"
        }
    ]
}
```

### Дисципліна (`subject.json`)
Файл дисципліни складається з масиву `layout`, який містить віджети:
```json
{
    "layout": [
        {
            "type": "hero",
            "title": "Заголовок",
            "subtitle": "Підзаголовок"
        }
    ]
}
```

## 5. Доступні віджети
| Тип віджета | Опис | Параметри |
| :--- | :--- | :--- |
| `hero` | Головний банер сторінки | `title`, `subtitle` |
| `info_card` | Інформаційний блок з виділенням | `title`, `content`, `variant` (neutral, info, success, warning, error) |
| `text_block` | Текстовий блок з підтримкою списків | `title`, `content` |
| `link_group` | Група посилань (список або кнопки) | `title`, `style` (list, buttons), `links` (label, url, description, icon) |
| `task_list` | Інтерактивний список завдань | `title`, `tasks` (id, text) |

## 6. Як додати нову дисципліну
1. Створіть новий JSON файл у папці `data/` (наприклад, `physics.json`).
2. Наповніть його контентом, використовуючи доступні віджети.
3. Додайте запис про нову дисципліну у `data/manifest.json`.

## 7. Локальний запуск
Оскільки додаток використовує JavaScript модулі та Fetch API, його потрібно запускати через локальний сервер:
- За допомогою розширення **Live Server** у VS Code.
- Або використовуючи Python: `python -m http.server 8000`.
- Або через Node.js: `npx serve .`.
